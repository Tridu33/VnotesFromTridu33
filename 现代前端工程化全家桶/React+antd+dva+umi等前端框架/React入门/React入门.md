







ReasonML就是OCaml，这意味着它的类型系统几乎和Haskell一样好，ReasonML类型系统最大的缺点是，缺乏类型类，不过它支持函子（更高阶的模型）。

ReasonML是静态类型的，它的类型推论几乎和Haskell一样好。


ReasonML编译成纯JS，因此可以在同一个项目中同时使用ReasonML和JS/TypeScript。
